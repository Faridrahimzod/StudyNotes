# STRIDE — Угрозы и контрмеры для Study Notes

| Поток/Элемент | Угроза (S/T/R/I/D/E) | Описание угрозы | Контроль | Ссылка на NFR | Проверка/Артефакт |
|---------------|-----------------------|------------------|----------|---------------|-------------------|
| F1 | S: Spoofing | Злоумышленник выдает себя за легитимного пользователя | Аутентификация с надежными паролями, многофакторная аутентификация | NFR-02, NFR-03 | Тесты аутентификации, мониторинг попыток входа |
| F1 | T: Tampering | Изменение данных в транзите между пользователем и API | HTTPS/TLS шифрование, проверка целостности данных | NFR-05 | SSL/TLS конфигурация, тесты на перехват данных |
| F1 | I: Information Disclosure | Перехват конфиденциальных данных (заметок) в транзите | Шифрование TLS, минимализация передаваемых данных | NFR-01, NFR-05 | Аудит передаваемых данных, проверка шифрования |
| F1 | D: Denial of Service | Перегрузка API большим количеством запросов | Rate limiting, ограничение размера запросов | NFR-03, NFR-10 | Нагрузочное тестирование, мониторинг ресурсов |
| F2 | S: Spoofing | Неавторизованный доступ к внутренним модулям | Внутренняя аутентификация, проверка прав доступа | NFR-02 | Тесты авторизации, аудит внутренних вызовов |
| F3 | R: Repudiation | Пользователь отрицает создание или изменение заметок | Подробное логирование, цифровая подпись операций | NFR-07 | Аудит логов, нефальсифицируемые записи |
| F3 | I: Information Disclosure | Утечка хэшей паролей из базы данных | Сильное хэширование (Argon2id), соление паролей | NFR-02 | Аудит безопасности БД, тесты на уязвимости |
| F4 | T: Tampering | Несанкционированное изменение заметок в БД | Контроль доступа, шифрование данных, целостность БД | NFR-01, NFR-04 | Тесты целостности данных, мониторинг изменений |
| F4 | I: Information Disclosure | Несанкционированный доступ к заметкам других пользователей | Строгая изоляция данных, проверка прав доступа | NFR-01, NFR-04 | Тесты авторизации, аудит запросов к БД |
| F4 | E: Elevation of Privilege | Получение прав администратора через уязвимости в API | Минимальные привилегии, проверка входных данных | NFR-04, NFR-06 | Тесты безопасности, сканирование уязвимостей |
| API | D: Denial of Service | Исчерпание ресурсов БД большим количеством запросов | Ограничение запросов, кэширование, мониторинг | NFR-03, NFR-10 | Нагрузочное тестирование, мониторинг БД |
| База данных | I: Information Disclosure | Прямой доступ к файлам БД минуя приложение | Шифрование БД, контроль доступа к файлам | NFR-01, NFR-09 | Аудит доступа к файлам, тесты безопасности |
